# dsa-patterns

Pattern-oriented data structures and algorithms practice in TypeScript, organized around the [NeetCode 150](https://neetcode.io/roadmap) roadmap.

The goal of this repository is not to accumulate solved problems. The goal is to build durable pattern recognition, defensible complexity analysis, and the communication habits required to perform under interview conditions.

## Workflow

### Daily

1. Pick a problem from the current pattern focus.
2. Solve from scratch in the appropriate `pattern/<pattern>/` folder using `templates/solution.template.ts`.
3. Write tests covering: a normal case, an edge case, an empty case, and at least one failure-prone case.
4. Run `yarn run test:watch` until green.
5. Fill in the solution header — including the mistake made and the complexity analysis.
